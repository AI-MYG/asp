# 单元完成上报对接说明（`/api/v1/progress/course-unit/complete`）

接口来自 PR 376。当一个**课程单元**（course unit）的最后一个游戏完成时，除了游戏包完成上报（`game-pack/complete`）之外，还要再调一次单元完成上报（`course-unit/complete`），**两者都成功才弹"完成当天课程" toast → 进入庆祝层**。

本文档面向后续接入"单元完成节点"游戏的开发者。已接入参考：

- `lib/ui/ket/game/vocabulary/sight_word_chant/ket_video_player_page.dart`
- `lib/ui/ket/game/vocabulary/sight_word_chant/ket_sight_word_chant_recording_page.dart`
- `lib/ui/ket/ket_game_video_page.dart`（仅 `gameTypeName == 'writing_grammar_board_game'` 时上报）

---

## 一、接口契约（PR 376）

| 项 | 内容 |
| --- | --- |
| 路径 | `POST /api/v1/progress/course-unit/complete` |
| 鉴权 | `Authorization: Bearer <JWT>`（沿用现有 `ApiService` 客户端） |
| Body | `{ "course_unit_id": "<UUID>" }` |
| 成功 | `code === 200`；`data` = `{ ok: true, course_unit_id, status: "completed" }` |
| 失败 | `code === 400`：`course_unit_id` 不存在或无效，message 含「course_unit_id 不存在或无效」 |
| 幂等 | 同一 `(用户, course_unit_id)` 重复上报后端会返回成功，**前端不需要做去重 / 防抖**；写入 `user_course_unit_game_completed`，`source=frontend` |

---

## 二、公共封装

所有上报逻辑都通过两个公共文件统一收口，**对接者不直接调 dio / ApiService**。

### 2.1 ApiService 方法

文件：`lib/data/api_service.dart`

```dart
Future<Map<String, dynamic>> reportCourseUnitComplete({
  required String courseUnitId,
});
```

- 失败由内部统一 `_handleUnifiedResponse` 抛 `ApiException`，无需自己处理 HTTP / code 判断。

### 2.2 上报工具：`CourseUnitCompleteReporter`

文件：`lib/common/game_celebration/game_pack_complete_reporter.dart`（与 game-pack 同文件，方便联用）

```dart
class CourseUnitCompleteReporter {
  static Future<bool> report({
    required String? courseUnitId,
    String tag = 'course-unit/complete',
  });
}
```

行为：

- **成功**（接口 `code===200`）→ 返回 `true`，调用方接着切庆祝层
- **失败 / `courseUnitId` 为空** → 自动 `Fluttertoast.showToast` 提示错误信息，返回 `false`
- 不做缓存、不做防抖（后端已幂等）

`tag` 用于日志区分游戏，例如 `'sight_word_chant'` / `'ket_game_video'`。

## 三、对接步骤

### 3.1 仅游戏包完成上报 vs. 兼调单元完成上报

| 场景 | 需要调用 |
| --- | --- |
| 普通游戏（非"单元最后一关"） | 只调 `GamePackCompleteReporter.report` |
| **单元最后一关**（如 sight_word_chant、writing_grammar_board_game） | **先 `GamePackCompleteReporter.report` → 成功后再 `CourseUnitCompleteReporter.report`** |

> **由产品/课程编排定义"哪个游戏是单元最后一关"**。代码侧通常通过 `gameTypeName` 或路由参数判断；不要凭空在所有游戏里都调单元上报，会造成单元数据偏早写入。

### 3.2 标准时序

```
玩家完成
   ↓
1) GamePackCompleteReporter.report(packId)         失败 → toast，不进庆祝
   ↓ 成功
2) CourseUnitCompleteReporter.report(courseUnitId) 失败 → toast，不进庆祝
   ↓ 成功
3) Toast: "完成当天课程"
   ↓
4) setState → 弹彩带→金币→星星
```

### 3.3 代码模板

```dart
import 'package:fluttertoast/fluttertoast.dart';
import 'package:inputbaby_app/common/game_celebration/game_pack_complete_reporter.dart';

Future<void> _enterFinishCelebration() async {
  if (_showFinishCelebration) return;

  // 1) 游戏包完成
  final bool gamePackOk = await GamePackCompleteReporter.report(
    packId: widget.packId,
    tag: 'my_game',
  );
  if (!mounted || !gamePackOk) return;

  // 2) 单元完成（仅"单元最后一关"调用；普通游戏跳过）
  final bool courseUnitOk = await CourseUnitCompleteReporter.report(
    courseUnitId: widget.courseUnitId,
    tag: 'my_game',
  );
  if (!mounted || !courseUnitOk) return;

  // 3) 提示 + 切庆祝
  Fluttertoast.showToast(
    msg: '完成当天课程',
    backgroundColor: Colors.white,
    textColor: Colors.black87,
    gravity: ToastGravity.CENTER,
  );
  setState(() => _showFinishCelebration = true);
}
```

参考实现：

- `ket_video_player_page.dart` 的 `_enterFinishCelebration`（sight_word_chant，全程必调单元上报）
- `ket_sight_word_chant_recording_page.dart` 的 `_enterCelebration`
- `ket_game_video_page.dart` 的 `_triggerCelebration`（按 `gameTypeName == 'writing_grammar_board_game'` 选择性调用）

## 四、`courseUnitId` 选取与透传规则

| 场景 | 来源 |
| --- | --- |
| 普通页 | `widget.courseUnitId` 由路由 query `courseUnitId` 透传 |
| 多级跳转的二级页（如视频页 → 录音页） | 上一级页面跳转时把 `courseUnitId` 拼进 URL 一并透传到下一级 |

> 参考：壳页 `KetSightWordChantPage._gotoGame` 把 `widget.courseUnitId` 拼进 `/ket/sight-word-chant-game?...&courseUnitId=...` 的 query，然后 `app.dart` 路由读出并传给 `KetVideoPlayerPage`。

## 五、错误与边界处理

| 场景 | 行为 |
| --- | --- |
| `game-pack/complete` 失败 | reporter 弹 toast，**第二步不调用**，不进庆祝 |
| `course-unit/complete` 失败 | reporter 弹 toast，**不进庆祝**（保护单元数据一致性） |
| `courseUnitId` 为空 / 空串 | reporter 弹 toast「完成上报失败：缺少 course_unit_id」，**不进庆祝** |
| 用户重复触发完成（动画重入等） | 后端幂等，前端**不需要**做防抖标记，允许重复调用 |
| 上报中用户离开页面 | 每个 `await` 后用 `if (!mounted) return;` 守卫，避免 setState 在已卸载组件上 |

> 与 `游戏完成上报对接说明.md` 完全一致：失败 toast，**不降级**展示庆祝。

## 六、Loading 处理

接口耗时通常很短（一次 INSERT），**不需要展示全屏 loading 蒙层**。当前规范是上报期间保持游戏当前画面即可。
